왜 구미 6반인가

합성 가능한 설계

모듈 간 결합도는 낮추고, 재사용성은 높인다. 컴포넌트 사고로 문제해결.

프레젠스-first

결과보다 과정, 기록보다 페르소나. 명확한 의사소통, 예쁜 화면.

리스크 헤지

테스트 먼저, 배포 먼저. 실패를 안전하게 대비하는 운영 원칙.

데이터 드리븐

가설 → 실험 → 피드백. 감으로 하지 않는다. 숫자로 말한다.

가독성 최우선

코드는詩, 이름은 설명서. 누가 봐도 읽히는 설계를 지향한다.

미래 친화적

Y2K 감성 × 모던 UX.技術과審美의 교집합을 만든다.

이 반의 타임라인

Orientation

목표 수립, 팀 identidade, 툴체인 세팅. 브랜딩 też.

Foundations

Web / Frontend

Backend / Server

API, DB, 캐시/큐, 보안, 스케일링, 이벤트驱动아키텍처.
